Detailed Description

template<typename FieldType, typename DerivedType, typename... Args>
class tensorwrapper::tensor::expression::detail_::NNary< FieldType, DerivedType, Args >

Code factorization for implementing ExpressionPIMPL derived classes.

Implementing a new derived class from ExpressionPIMPL requires overriding several virtual methods. The implementations for many of these methods are largely boilerplate. Using the curiously recursive template pattern, the NNary class implements these methods for the derived class.

The name NNary comes from the generalizaton of unary, binary, trinary, etc. and refers to the fact that this class can handle implementing expressions involving an arbitrary number of arguments (although in practice only unary and binary versions are presently encountered, it was easy enough to write this class in a fully general manner).

Template Parameters
FieldTypeA strong type denoting whether the elements of the associated tensors are scalars or other tensors. Expected to be either field::Scalar or field::Tensor.
DerivedTypeThis is the type of the class (including any template parameters) being implemented by NNary. For example for the Add template class this would be Add<FieldType>.
Args...A variadiac template parameter pack whose members are the types of the pieces of the derived expression. For example a binary expression which combines two Expression instances would set Args to Expression, Expression.

Implements are_equal.

This method is implemented by attempting to downcast rhs to my_type. If the cast succeeds, the m_args_ members are then compared. Of note, this means that if the derived class contains additional state not exposed to the NNary base class, the derived class must also override are_equal.

Parameters
[in]rhsThe expression we are polymorphically comparing to.
Returns
True if rhs can be downcast to my_type and if this->m_args_ compares equal to rhs.m_args_. False otherwise.
Exceptions
NoneNo throw guarantee.
