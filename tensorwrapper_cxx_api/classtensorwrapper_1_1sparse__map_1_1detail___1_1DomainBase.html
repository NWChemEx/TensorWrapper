Detailed Description

template<typename DerivedType, typename IndexType>
class tensorwrapper::sparse_map::detail_::DomainBase< DerivedType, IndexType >

DomainBase contains functionality common to Domain<ElementIndex> and Domain<TileIndex>.

Template class specialization requires you to redefine all methods for each specialization. To avoid excessive copy/paste we siphon off methods common to Domain<ElementIndex> and Domain<TileIndex> and put them into the DomainBase class. DomainBase instances are not meant to be directly instantiated, rather users should always declare either a Domain<ElementIndex> or a Domain<TileIndex> instance.

Template Parameters
DerivedTypeThe type of the Domain using this class as a base class. Parts of DomainBase work by CRTP hence the need to know what class is being implemented.
IndexTypeThe type of the index in the Domain. Should be the same as typename DerivedType::value_type.
Note
This class only implements operations among Domains which store the same index types. Operations among Domains with different index types must be implemented in the derived classes.
Domains without PIMPLs behave as if they are empty.

◆ inject()

template<typename DerivedType , typename IndexType >
DerivedType tensorwrapper::sparse_map::detail_::DomainBase< DerivedType, IndexType >::inject ( const std::map< size_type, size_type > &  injections) const

Creates a new domain by injecting a pinned index.

Sometimes, when sparsifying tensors, modes of the tensor are actually independent indices. In this case the domains contain dependent indices corresponding to slices of the tensor. To get a full index for the tensor we need to inject one (or more) modes of an independent index into each index in the domain. For example, assume the independent index 0 maps to dependent indices 1, 2, and 3. When sparsifying a rank 2 tensor where the independent index is mode 1 (0-based) we need to extract from the tensor elements (1, 0), (2, 0), and (3, 0), i.e., we need to "inject" 0 into each index so that it is value of mode 1.

Note
Injecting into an empty Domain always results in an empty Domain.
Since the size of injections is partly used to determine the rank of the resulting Domain, injections can not contain any keys outside the rank [0, rank() + injections.size()].
Parameters
[in]injectionsA map such that injections[i] is the value of the i-th mode of each index in the resulting Domain.
Returns
A Domain containing the indices formed by injecting from injections
Exceptions
std::out_of_rangeif the Domain is non-empty and any of the keys in injections is not in the range [0, rank()
  • injections.size()]. Strong throw guarantee.
std::bad_allocif there is insufficient memory to create the new Domain. Strong throw guarantee.
